web application development fast and easy.with the help of it we can develop our application very and easily and fastly.
-->Advantages:
1) It is javascript development so fast and easy.
2)Easy to configure and customize.
3)We work on middlewares so the application work is easy.

const express=require('express'); //import
const PORT=8899;// define PORT//

// define the routes

->configuring routes(routing)

routing refers to determining how an applcaiton responds to a client request to a particular endpoint, which
is URI (or path ) and a specific HTTP request method
syntax:
app.METHOD(PATH,HANDLER)
app: instance of express
-->METHOD: http request methods(GET,POST,PUT,DELETE)
=>PATH: URI
->HANDLER: function executed when path is match
eg:
app.get('/',(req,res)=>{
    res.send("default")
})
app.post('/submit-data',(req,res)=>{
    res.send('post request');

})

app.put('/update-data',(req,res)=>{
    res.send('put request');
    
})
app.delete('/delete-data',(req,res)=>{
    res.send('delete request');
    
})

=>express js middleware

->middleware are the functions that execute during the lifecycle of a request to the server.
->Each middleware has access to the http request and response for each route(path).
->Types of middleware:
1)custom middleware

let myData=(req,res,next)=>{
    req.mytime="Neosoft Technology"
    next();


}
->To use any middleware we have .use()
eg:
app.use(myData())

->express.Router:
It is a inbuild middleware which is used to create separate route handlers.

routes/adminRoutes.js

const express=require('express');
const router=express();
router.get('/',(req,res)=>{
    res.send("admin front")
})
router.get('/dashboard',(req,res)=>{
    res.send("admin dashboard ")
})
module.exports=router;

app.js


const frontRoutes=require('./routes/frontRoutes')
const adminRoutes=require('./routes/adminRoutes')
app.use('/',frontRoutes)
app.use("/admin",adminRoutes);
